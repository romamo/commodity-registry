## 1. Goals

- Prevent user-defined instruments from colliding with standard currency names (like EUR, USD).
- Enrich the registry with additional identifying metadata (Country, IBKR conid) and price tickers for currencies.
- Ensure the core registry model only contains persistent, constant identifying data.

## 2. Work Done

- Created `src/instrument_registry/data/instruments/currencies.yaml` with canonical `Cash` entries for major currencies (EUR, USD, GBP, CHF, JPY) and their Yahoo Finance tickers (e.g., `EURUSD=X`).
- Updated `add_instrument` in `registry.py` and the CLI `add` command to check for cross-asset-class name collisions before adding a new item.
- Expanded `Instrument` and `SearchResult` models to include a persistent `country` field and a generic `metadata` dictionary for flexible extensibility.
- Removed non-persistent, subjective fields (`risk_profile`, `liquidity`) and their associated Enums from the core `Instrument` model.
- Added `ibkr` (Interactive Brokers conid) as a supported integer field in the `Tickers` model.
- Updated the `resolve` CLI command to automatically detect numeric strings and resolve them as IBKR conids.

## 3. Decisions Made

| Decision | Rationale |
|----------|-----------|
| Add canonical currencies to bundled data | Prevents users from accidentally adding a stock or ETF with the name "EUR" or "USD" by triggering the name collision check. |
| Add `country` and `metadata` to models | `country` is a constant property useful for geo-allocation. `metadata` provides a flexible way to store arbitrary static facts without altering the Python models. |
| Add `ibkr` to `Tickers` model | IBKR conids are immutable, numeric, exchange-specific identifiers that perfectly align with the registry's goal of being a "Single Source of Truth." |

## 4. Decisions Refused / Alternatives Rejected

| Rejected Approach | Reason |
|-------------------|--------|
| Moving `risk_profile` and `liquidity` into `metadata` | The registry is designed for constant, persistent data. These fields are dynamic/subjective and thus were removed entirely rather than just relocated. |
